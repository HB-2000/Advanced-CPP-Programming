تمرین پیاده‌سازی کلاس: سیستم مدیریت اطلاعات دانشجو

این تمرین برای تقویت مفاهیم برنامه‌نویسی شیءگرا و پیاده‌سازی کلاس در زبان C++ طراحی شده است.

  • محدودیت زمان: ۱ ثانیه
  • محدودیت حافظه: ۲۵۶ مگابایت

شرح مسئله

شما باید یک سیستم ساده برای مدیریت اطلاعات یک دانشجو پیاده‌سازی کنید. این سیستم شامل یک کلاس به نام Student است که اطلاعات دانشجو (نام، شماره دانشجویی و معدل) را ذخیره کرده و امکان انجام عملیات مختلفی مانند به‌روزرسانی معدل، بررسی وضعیت قبولی، افزودن جایزه به معدل، بازنشانی اطلاعات و مقایسه معدل را فراهم می‌کند.

مشخصات کلاس Student

ویژگی‌ها

  • name: نام دانشجو (رشته، بدون فاصله، حداکثر ۱۰۰ کاراکتر).
  • studentId: شماره دانشجویی (عدد صحیح مثبت).
  • gpa: معدل دانشجو (عدد اعشاری در بازه‌ی [0, 20]).

سازنده

سازنده‌ای که سه پارامتر دریافت کند:

  • name (رشته): نام دانشجو.
  • studentId (عدد صحیح): شماره دانشجویی.
  • gpa (عدد اعشاری): معدل اولیه.
  • این مقادیر باید ویژگی‌های کلاس را مقداردهی کنند.

مخرب

مخربی که هنگام حذف شیء، پیام زیر را چاپ کند:

Student with ID X is being deleted.

که در آن X شماره دانشجویی است.

متدهای Getter

  • getName(): مقدار name را برگرداند.
  • getStudentId(): مقدار studentId را برگرداند.
  • getGpa(): مقدار gpa را برگرداند.

متدهای Setter

  • setName(string n): مقدار name را به n تغییر دهد.
  • setGpa(double g): مقدار gpa را به g تغییر دهد، مشروط بر اینکه 0 ≤ g ≤ 20. در صورت معتبر بودن، پیام زیر چاپ شود: 
    GPA updated to X.
    که X معدل جدید با ۲ رقم اعشار است. در غیر این صورت، پیام زیر چاپ شود: 
    Invalid GPA value.

سایر متدها

  1. updateGpa(double newGpa):

    معدل را به newGpa تغییر دهد (با استفاده از منطق setGpa برای اعتبارسنجی و چاپ پیام).

  2. isPassing():

    اگر gpa ≥ 12 باشد، پیام زیر چاپ شود:

    Student is passing.

    در غیر این صورت:

    Student is failing.
  3. addBonusGpa(double bonus):

    مقدار bonus را به gpa اضافه کند، مشروط بر اینکه:

    • bonus ≥ 0
    • gpa + bonus ≤ 20

    در صورت معتبر بودن، پیام زیر چاپ شود:

    Bonus added, new GPA is X.

    که X معدل جدید با ۲ رقم اعشار است. در غیر این صورت:

    Invalid bonus value or exceeds GPA limit.
  4. resetStudent():

    مقدار name را به "Unknown" و gpa را به 0 تغییر دهد و پیام زیر را چاپ کند:

    Student data reset, GPA is 0.
  5. compareGpa(double value):

    معدل را با value مقایسه کند و یکی از پیام‌های زیر را چاپ کند:

    • اگر gpa < value: 
      GPA is less than X.
    • اگر gpa = value: 
      GPA is equal to X.
    • اگر gpa > value: 
      GPA is greater than X.

    که X مقدار value با ۲ رقم اعشار است.

وظیفه شما

برنامه‌ای بنویسید که:

  1. کلاس Student را با مشخصات بالا پیاده‌سازی کند.
  2. اطلاعات اولیه‌ی یک دانشجو (نام، شماره دانشجویی و معدل) را از ورودی بخواند و یک شیء از کلاس Student ایجاد کند.
  3. تعداد عملیات (q) را از ورودی بخواند.
  4. به تعداد q عملیات، دستورات را از ورودی بخواند و متد مربوطه را اجرا کند.
  5. برای هر عملیات، پیام مناسب چاپ شود.
  6. در انتها، پیام مخرب هنگام حذف شیء چاپ شود.

فرمت ورودی

ورودی شامل چند خط است:

  • خط اول:

    سه مقدار جدا شده با فاصله:

    • نام دانشجو (رشته بدون فاصله).
    • شماره دانشجویی (عدد صحیح).
    • معدل اولیه (عدد اعشاری).
  • خط دوم:

    یک عدد صحیح q (تعداد عملیات).

  • q خط بعدی:

    هر خط یک عملیات را مشخص می‌کند:

    • عملیات ۱: عدد 1 و یک عدد اعشاری newGpa (اجرای updateGpa(newGpa)).
    • عملیات ۲: عدد 2 (اجرای isPassing()).
    • عملیات ۳: عدد 3 و یک عدد اعشاری bonus (اجرای addBonusGpa(bonus)).
    • عملیات ۴: عدد 4 (اجرای resetStudent()).
    • عملیات ۵: عدد 5 و یک عدد اعشاری value (اجرای compareGpa(value)).

محدودیت‌ها

  • 1 ≤ studentId ≤ 10^6
  • 0 ≤ gpa, newGpa, bonus, value ≤ 20
  • 1 ≤ q ≤ 100
  • نام دانشجو رشته‌ای بدون فاصله و حداکثر ۱۰۰ کاراکتر است.
  • مقادیر اعشاری ممکن است تا ۶ رقم اعشار داشته باشند.

فرمت خروجی

برای هر عملیات، پیام مرتبط با متد مربوطه چاپ شود:

  • عملیات ۱ (updateGpa):

    اگر معتبر باشد: GPA updated to X. (X معدل جدید با ۲ رقم اعشار).

    در غیر این صورت: Invalid GPA value.

  • عملیات ۲ (isPassing):

    اگر gpa ≥ 12: Student is passing.

    در غیر این صورت: Student is failing.

  • عملیات ۳ (addBonusGpa):

    اگر معتبر باشد: Bonus added, new GPA is X. (X معدل جدید با ۲ رقم اعشار).

    در غیر این صورت: Invalid bonus value or exceeds GPA limit.

  • عملیات ۴ (resetStudent):

    همیشه: Student data reset, GPA is 0.

  • عملیات ۵ (compareGpa):

    بسته به مقایسه: GPA is less than X.، GPA is equal to X.، یا GPA is greater than X. (X مقدار ورودی با ۲ رقم اعشار).

  • در انتها:

    پیام مخرب: Student with ID X is being deleted. (X شماره دانشجویی).

تمام اعداد اعشاری در خروجی باید با دقیقاً ۲ رقم اعشار چاپ شوند.

مثال‌ها

مثال ۱

ورودی:

Ali 12345 15.5
5
1 16.75
2
3 1.5
4
5 10.0

خروجی:

GPA updated to 16.75.
Student is passing.
Bonus added, new GPA is 18.25.
Student data reset, GPA is 0.
GPA is less than 10.00.
Student with ID 12345 is being deleted.

توضیحات:

  • دانشجو با نام Ali، شماره دانشجویی 12345 و معدل 15.5 ایجاد می‌شود.
  • عملیات‌ها:
    1. معدل به 16.75 به‌روزرسانی می‌شود.
    2. معدل 16.75 بررسی می‌شود (قبول).
    3. جایزه 1.5 اضافه می‌شود، معدل به 18.25 می‌رسد.
    4. اطلاعات بازنشانی می‌شود (name = "Unknown"، gpa = 0).
    5. معدل 0 با 10.0 مقایسه می‌شود (کمتر است).
  • در انتها، پیام مخرب چاپ می‌شود.

مثال ۲

ورودی:

Sara 67890 11.0
4
3 10.0
2
1 21.0
5 11.0

خروجی:

Invalid bonus value or exceeds GPA limit.
Student is failing.
Invalid GPA value.
GPA is equal to 11.00.
Student with ID 67890 is being deleted.

توضیحات:

  • دانشجو با نام Sara، شماره دانشجویی 67890 و معدل 11.0 ایجاد می‌شود.
  • عملیات‌ها:
    1. جایزه 10.0 نامعتبر است (معدل به 21.0 می‌رسد).
    2. معدل 11.0 بررسی می‌شود (مردود).
    3. معدل 21.0 نامعتبر است.
    4. معدل 11.0 با 11.0 مقایسه می‌شود (برابر).
  • در انتها، پیام مخرب چاپ می‌شود.

نکات مهم

  • از استاندارد C++11 یا بالاتر استفاده کنید.
  • برای چاپ اعداد اعشاری، از fixed و setprecision(2) استفاده کنید.
  • ورودی‌ها معتبر فرض می‌شوند (مثلاً نام بدون فاصله، مقادیر در محدوده‌ی مشخص).
  • پیام مخرب باید در انتهای خروجی و پس از تمام عملیات‌ها چاپ شود.
  • متدها باید دقیقاً پیام‌های مشخص‌شده را چاپ کنند (بدون فضای اضافی یا تغییرات در متن).
  • نیازی به اعتبارسنجی مقادیر ورودی فراتر از آنچه در متدها مشخص شده نیست (مثلاً نیازی به بررسی منفی بودن studentId نیست).
